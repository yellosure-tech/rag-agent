# RAG 与 Agent 笔记

RAG 是 Retrieval-Augmented Generation 的缩写，即检索增强生成。它将外部知识库检索到的相关上下文提供给大语言模型，降低模型在专业文档问答中的幻觉问题。

## 基本流程

一个最小 RAG 系统通常包含文档读取、文本切分、向量化、相似度检索、上下文拼接和回答生成。文本切分需要设置 chunk size 和 overlap，避免重要信息被切断。检索阶段通常返回 top-k 个相关片段，并在回答中附带来源引用。

## Agent 工具调用

Tool Use Agent 会根据用户意图选择工具。例如，知识性问题可以调用 `search_docs` 检索文档；计算问题可以调用 `calculator`；总结类问题可以调用 `summarize_doc`。工具调用失败时，需要记录失败原因，例如参数解析错误、检索为空或上下文不足。

## 评测方法

RAG 项目可以构建一组测试问题，对比无 RAG 和有 RAG 的回答。常见评估维度包括召回是否命中、答案是否有来源、是否减少幻觉、上下文是否足够和回答是否覆盖关键事实。
